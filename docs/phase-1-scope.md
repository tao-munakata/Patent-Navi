# Phase 1 スコープ

## 1. 目的

Phase 1 では、AI API やデータベースには接続せず、特許検索ナビサイトの基本体験を静的プロトタイプで確認する。

目的は、販売用 MVP の前に以下を検証することである。

- 入力項目がユーザーにとって分かりやすいか
- 調査プランの表示順が自然か
- キーワード、調査軸、検索リンクが実務で使えそうか
- 請求項貼り付け分析の流れが理解しやすいか
- Markdown エクスポートが相談前メモとして使えるか

## 2. Phase 1 に含めるもの

| 機能 | 内容 |
| --- | --- |
| アイデア入力 | 機能概要、課題、利用シーン、部品・技術、競合情報を入力できる |
| 調査プラン生成 | サンプルロジックでキーワード、調査軸、チェックリストを生成する |
| 検索リンク生成 | Google Patents、Espacenet、J-PlatPat への導線を表示する |
| 請求項分析メモ | 貼り付けた請求項を構成要素と回避設計案に整理する |
| Markdown 出力 | 入力、検索式、分析メモを Markdown として出力する |
| 免責表示 | 法的判断ではないことを画面内に表示する |

## 3. Phase 1 に含めないもの

- 本物の AI API 接続
- ユーザー認証
- サーバー保存
- 調査履歴
- 課金
- 外部検索サイトの結果取得
- PDF や図面のアップロード解析
- パテントマップ
- 法的判断、鑑定、侵害有無の断定

## 4. 完了条件

Phase 1 は以下を満たせば完了とする。

- ブラウザで `prototype/index.html` を開ける
- 初期入力例から調査プランを生成できる
- キーワード、調査軸、検索リンク、チェックリストを確認できる
- Google Patents、Espacenet、J-PlatPat の導線が表示される
- 請求項分析メモで構成要素と回避設計案が表示される
- Markdown 出力が実行できる
- 画面上に免責表示がある

## 5. 次フェーズへの引き継ぎ

Phase 2 では、この静的プロトタイプを Next.js + TypeScript に移し、AI API 接続を行う。

優先する作業:

1. Next.js アプリ初期化
2. 画面コンポーネント化
3. 検索リンク生成ロジックの `lib` 化
4. AI 出力 JSON Schema の型定義化
5. `/api/generate-plan` の実装
6. エラー処理と出力バリデーション
