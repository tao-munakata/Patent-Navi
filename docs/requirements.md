# 特許検索ナビサイト 要件定義

## 1. 目的

本システムは、専門的な特許検索知識を持たない事業企画者、開発者、経営者が、事業・製品アイデアから関連特許を探すための一次調査支援サイトである。

特許検索サービスそのものを置き換えるのではなく、ユーザーの自然文入力を、特許検索に適した技術キーワード、作用・動作、機能的表現、用途観点、分類観点へ変換し、J-PlatPat、Google Patents、Espacenet などの主要検索サイトへ橋渡しする。

本システムの位置づけは、弁理士相談や本格的な FTO 調査に進む前の一次フィルタリングである。

## 2. 想定ユーザー

| ユーザー | 主な目的 | 期待する価値 |
| --- | --- | --- |
| スタートアップ創業者 | 新規事業アイデアの特許リスクを早期確認したい | 専門用語を知らなくても調査の入口に立てる |
| 事業開発担当者 | 新製品・新サービスの周辺特許を把握したい | 競合・用途・技術軸で漏れを減らせる |
| 開発者・PdM | 実装予定機能が他社特許に近いか確認したい | 回避設計の検討材料を得られる |
| 中小企業の知財担当 | 外部専門家に依頼する前に論点を整理したい | 相談前の下調べとメモを作れる |

## 3. 提供価値

- ビジネスアイデアを特許検索向けの言葉に変換する
- 検索漏れを減らすための多角的な調査軸を提示する
- 主要検索サイトにすぐ遷移できる検索式付きリンクを生成する
- 検索結果や請求項を貼り付け、侵害リスク確認の観点を整理する
- 回避設計、競合分析、相談用メモ作成を支援する

## 4. MVP スコープ

初期販売可能な MVP では、以下を実装対象とする。

| 区分 | MVP に含める | MVP では含めない |
| --- | --- | --- |
| 入力 | アイデア、課題、業界、部品、競合企業の入力 | 図面アップロード、音声入力 |
| AI 生成 | キーワード、類義語、作用・動作、機能的表現、用途軸 | 完全自動の法的判断 |
| 検索リンク | Google Patents、J-PlatPat、Espacenet へのリンク生成 | 各検索サイトの結果スクレイピング |
| 分析 | 貼り付けた請求項・要約に対する観点整理 | 権利有効性の断定、侵害有無の断定 |
| 保存 | Markdown エクスポート | チーム共有、監査ログ、案件管理 |

## 5. 機能要件

### 5.1 アイデア入力

ユーザーは以下の項目を入力できる。

| 項目 | 必須 | 用途 |
| --- | --- | --- |
| 機能概要 | 必須 | 作用・動作、技術キーワード抽出の基礎 |
| 解決したい課題 | 推奨 | 目的・効果、F ターム観点の抽出 |
| 利用シーン・業界 | 推奨 | 用途特許、異業種展開の探索 |
| 具体的な部品・技術 | 任意 | 構成要素、回避設計観点の整理 |
| 競合製品・企業 | 任意 | 出願人検索、競合分析の起点 |

### 5.2 AI による多角的キーワード生成

システムは入力内容から以下を生成する。

- 日本語キーワード
- 英語キーワード
- カタカナ・略語・表記ゆれ
- 同義語・類義語
- 作用・動作ベースの表現
- 特許で使われやすい機能的表現
- 除外した方がよい曖昧語

生成結果には、なぜそのキーワードが有効かを短く表示する。

### 5.3 調査軸の自動提案

システムは以下の観点を提案する。

- 技術構成から探す軸
- 解決課題から探す軸
- 作用・動作から探す軸
- 用途・業界から探す軸
- 競合企業・出願人から探す軸
- 回避設計のために確認すべき構成要素

F タームや FI については、MVP では「候補観点の提示」までとし、正確な分類コードの断定は避ける。

### 5.4 検索リンク生成

システムは、生成した検索式をもとに以下の外部サイトへのリンクを作成する。

| サイト | 主な用途 |
| --- | --- |
| Google Patents | 技術キーワードによるグローバル検索、図面確認 |
| J-PlatPat | 日本国内の権利状況、FI/F ターム調査 |
| Espacenet | 欧州、米国、中国などの海外特許調査 |

MVP では、検索結果の自動取得は行わず、外部サイトを新規タブで開く導線とする。

### 5.5 検索結果の貼り付け分析

ユーザーは、外部サイトで見つけた特許の要約、請求項、出願人、公開番号などを貼り付けられる。

システムは以下を支援する。

- 請求項の構成要素分解
- 自社アイデアとの対応表作成
- オールエレメントルール確認用チェックリスト作成
- 不明点、専門家に確認すべき論点の抽出

出力では「侵害している」「侵害していない」と断定しない。

### 5.6 回避設計アイデア生成

システムは、貼り付けられた請求項または構成要素に対して、以下の方向から回避案を提示する。

- 構成要素の削除
- 構成要素の置換
- 処理順序の変更
- 使用条件・用途の限定
- 入力・出力・制御方式の変更
- 物理構造からソフトウェア制御への置換、またはその逆

各案には、実現難易度、事業影響、再調査が必要な観点を付ける。

### 5.7 競合分析支援

競合企業名や出願人情報をもとに、以下を整理する。

- 調査すべき出願人名の表記ゆれ
- 関連しそうな技術領域
- 重点的に見るべき年代、国、用途
- 競合の製品戦略を推測するための観点

MVP では、実際の出願件数集計や自動パテントマップは対象外とする。

### 5.8 Markdown エクスポート

ユーザーは以下を Markdown 形式でエクスポートできる。

- 入力内容
- 生成キーワード
- 検索式
- 外部検索リンク
- 分析メモ
- 回避設計案
- 専門家に相談すべき論点

## 6. 画面要件

### 6.1 トップ画面

トップ画面は説明中心のランディングページではなく、すぐに調査を開始できる入力画面とする。

主な構成:

- アイデア入力フォーム
- 入力例の表示
- 調査開始ボタン
- 免責と用途の短い説明

### 6.2 調査結果画面

主な構成:

- キーワード候補
- 調査軸
- 検索式
- 外部検索リンク
- 次に確認すべきチェックリスト

### 6.3 分析画面

主な構成:

- 検索結果・請求項の貼り付け欄
- 構成要素分解
- 自社アイデアとの対応表
- リスク確認チェックリスト
- 回避設計案
- Markdown エクスポートボタン

## 7. 非機能要件

| 項目 | 要件 |
| --- | --- |
| 販売前提 | 法的判断ではなく調査支援であることを明示する |
| セキュリティ | 入力された未公開アイデアや競合情報を慎重に扱う |
| プライバシー | 保存範囲、AI 利用範囲、ログ保持方針を明示する |
| 応答速度 | キーワード生成は 15 秒以内を目標とする |
| 可用性 | MVP では単一アプリ構成でよいが、外部 AI API 障害時のエラー表示を用意する |
| 監査性 | 有料版では調査履歴、出力日時、使用モデルを記録できる設計にする |
| 多言語 | MVP は日本語 UI、検索語は日本語・英語対応 |

## 8. 法務・表示上の注意

本システムは法律相談、鑑定、侵害判断、無効判断、権利有効性判断を提供しない。

画面上には以下の趣旨を明示する。

- AI 出力は調査の観点整理であり、法的判断ではない
- 最終判断は弁理士・弁護士などの専門家に相談する必要がある
- 外部検索サイトの検索結果や権利状態は必ず原典で確認する
- 未公開発明や営業秘密を入力する場合のリスクをユーザーに説明する

## 10. AI プロンプト設計方針

AI には以下の制約を与える。

- 法的結論を断定しない
- 特許調査の観点、検索語、確認項目として出力する
- ユーザーの入力が曖昧な場合は、仮定を明示する
- 構成要素、作用、用途、課題を分けて整理する
- 出力は JSON など構造化データで受け取り、UI 側で表示する
